N1 — Diagrama de Containers

Decisões de design

DecisãoMotivo
Native SQL em vez de JPQLAs queries do BI têm custo D$64 de 261K — precisam ser exatas para garantir os mesmos resultados. JPQL não consegue expressar os CASE Pronampe Solidário RS, subqueries de DT_REF=MAX nem EVT_OPR_R3TD.
Dois schemas DB2 (GFG + D4W)DB2GFG é transacional (atualizado em tempo real). DB2D4W é o Data Warehouse pré-computado (snapshot diário). Operações e pendências vêm do DW por performance.
Redis apenas para warm-up de KPIsOs 22 endpoints v1 são queries diretas no DB2 — suficientemente rápidos para o volume de agentes. Redis é reservado para KPIs agregados de alta frequência.
DT_REF = MAX subquery em todas as queries de CTRA_FNDO_GRTRA tabela guarda histórico de snapshots. O filtro garante que sempre vemos o estado mais recente da carteira, sem precisar deletar dados antigos.
CD_FNDO_GRTR <> 1FGO Original (código 1) é excluído conforme padrão do BI — não está no escopo do Painel do Agente.
